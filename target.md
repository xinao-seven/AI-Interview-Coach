
**AI Interview Coach：面向前端 / WebGIS 求职者的智能面试训练与简历项目助手**

核心功能可以拆成下面几大模块。

---

## 一、简历解析模块

这是整个项目的入口。用户上传简历后，系统自动提取简历中的关键信息，为后面的面试题生成和项目追问做准备。

需要实现的功能包括：

支持上传 PDF、Word 或 Markdown 简历。第一版可以先支持 PDF 或纯文本，降低难度。

解析简历中的基本信息，例如姓名、求职方向、教育经历、技术栈、项目经历、实习经历、获奖经历等。

把简历内容结构化展示出来，例如：

```ts
{
  name: "xxx",
  targetRole: "前端开发 / WebGIS开发",
  skills: ["Vue3", "TypeScript", "OpenLayers", "Cesium", "ECharts"],
  projects: [
    {
      name: "三维地质模型可视化平台",
      techStack: ["Vue3", "Three.js", "Cesium"],
      description: "...",
      highlights: ["钻孔数据建模", "三维模型加载", "剖切分析"]
    }
  ]
}
```

前端展示上可以做成“左侧原始简历预览，右侧结构化解析结果”。用户可以手动修改 AI 解析错误的内容。

这个模块的亮点是：**AI 不是直接聊天，而是先把用户简历转成结构化数据，后续所有问题都基于这个结构化数据生成。**

---

## 二、岗位与面试模式选择模块

用户进入训练前，需要先选择目标岗位和训练模式。

岗位方向可以包括：

前端开发工程师
WebGIS 开发工程师
Vue 开发工程师
AI 前端应用开发工程师
实习 / 校招前端岗位

面试模式可以包括：

技术基础面试
项目深挖面试
八股文训练
场景题训练
综合模拟面试
压力面试

难度可以分为：

简单：偏基础概念，例如 Vue、CSS、JS、HTTP。
中等：结合项目提问，例如“你的项目中 Pinia 是怎么设计的？”
困难：偏真实面试追问，例如“如果地图上有十万个点，怎么优化渲染？”

这个模块建议做成一个配置面板，用户选择完成后点击“开始面试”。

---

## 三、AI 面试题生成模块

这是项目的核心之一。系统根据简历内容、岗位方向和面试模式生成问题。

题目来源可以分为三类：

第一类是通用前端基础题，例如：

Vue3 的响应式原理是什么？
Composition API 相比 Options API 有什么优势？
Pinia 和 Vuex 有什么区别？
HTTP/1.1 和 HTTP/2 的区别是什么？
SSE 和 WebSocket 有什么区别？

第二类是项目相关问题，例如：

你在三维地质模型可视化平台中负责了哪些模块？
为什么 Cesium 适合做地理场景，而 Three.js 更适合做模型展示？
钻孔 Excel 数据是如何解析并映射到三维场景中的？
ECharts 图表和地图点击事件是如何联动的？
如果模型加载很慢，你会从哪些方面优化？

第三类是追问型问题，例如：

你刚才提到用了 Pinia，那为什么没有直接用 reactive 全局对象？
你说做了图层控制，如果图层很多，状态结构怎么设计？
你提到使用 SSE 实现 AI 流式输出，如果连接中断怎么处理？

前端上不要一次性展示所有题目，可以像真实面试一样，一个问题一个问题地问。这样更有交互感。

---

## 四、模拟面试对话模块

这是用户真正使用最多的页面。

基本流程是：

AI 面试官提出问题。
用户输入回答，可以先做文本输入，后续再扩展语音输入。
用户提交回答。
AI 对回答进行评价。
AI 根据回答继续追问。
一轮结束后进入下一个问题。

页面结构可以设计成类似聊天界面，但要比普通聊天更专业。

建议界面包括：

左侧：当前面试进度，例如第 3 / 10 题。
中间：AI 面试官与用户的对话。
右侧：当前题目的考察点、用户得分、改进建议。

回答输入框可以支持：

普通文本输入
Markdown 输入
代码片段输入
一键“我不会”
一键“给我提示”
一键“查看参考答案”

这里可以体现你的前端能力，比如 SSE 流式输出、Markdown 渲染、代码高亮、滚动控制、消息状态管理、错误重试等。

---

## 五、回答评分与反馈模块

用户每回答一个问题，系统应该给出结构化反馈，而不是只说“回答得不错”。

建议评分维度包括：

准确性：概念是否正确。
完整性：是否覆盖关键点。
表达能力：是否适合面试表达。
项目结合度：是否结合自己的项目经历。
深度：是否能回答追问和底层原理。

可以显示成评分卡：

```ts
{
  totalScore: 78,
  accuracy: 80,
  completeness: 75,
  expression: 85,
  projectRelevance: 70,
  depth: 65,
  strengths: ["能说明基本概念", "表达比较清晰"],
  weaknesses: ["缺少项目中的具体例子", "没有解释底层原理"],
  improvedAnswer: "更好的回答可以是..."
}
```

前端可以用 ECharts 做一个雷达图，展示用户在不同维度上的表现。

这个模块很重要，因为它能让项目从“聊天机器人”升级为“训练系统”。

## 六、项目深挖助手模块

这个模块最适合突出你的特色。很多面试官都会围绕简历项目不断追问，所以你可以单独做一个“项目深挖”页面。

用户选择简历中的某个项目，比如：

“三维地质模型可视化平台”

系统自动生成几个部分：

项目介绍优化
技术亮点提炼
可能被问到的问题
高频追问
项目难点回答模板
STAR 法则回答
面试版 1 分钟介绍
面试版 3 分钟介绍

例如项目深挖可以生成：

“这个项目的核心难点是什么？”
“为什么选择 Three.js 而不是 Cesium？”
“钻孔 Excel 数据如何转成三维模型？”
“如何处理模型加载性能问题？”
“如果让你重新设计这个项目，你会怎么优化？”

这个功能非常实用，也很适合你自己准备面试。

## 八、面试报告生成模块

一次模拟面试结束后，系统生成一份完整报告。

报告内容包括：

本次面试岗位
面试题数量
总评分
优势总结
薄弱知识点
项目表达问题
推荐复习内容
每道题的回答记录
每道题的参考答案
下一次训练建议

可以生成类似这样的结论：

“你在 Vue3 基础和项目描述方面表现较好，但在性能优化、浏览器原理和 WebGIS 大数据渲染方面回答不够深入。建议后续重点训练地图大数据渲染、SSE 错误处理、Pinia 状态设计和项目难点表达。”

这个模块可以支持导出 Markdown 或 PDF。第一版可以先导出 Markdown，后续再做 PDF。

## 九、历史记录与错题本模块

如果你想让项目更像一个完整产品，可以加入历史记录。

需要保存：

每次模拟面试记录
每道题的用户回答
AI 评分
AI 追问
用户收藏的问题
用户薄弱知识点

错题本可以按知识点分类，例如：

Vue3：3 道薄弱题
HTTP：5 道薄弱题
WebGIS：4 道薄弱题
项目表达：6 道薄弱题

这样可以形成闭环：

简历解析 → 生成问题 → 模拟面试 → 评分反馈 → 生成报告 → 错题复习 → 再次训练。

## 十、AI 简历优化模块

既然项目叫“简历项目助手”，可以加入简历优化功能。

具体功能包括：

简历问题诊断
项目描述润色
技术亮点提炼
量化表达优化
根据岗位 JD 匹配简历
生成面试自我介绍
生成项目介绍话术

比如用户上传一段项目描述：

“使用 Vue3 和 Three.js 实现三维地质模型展示。”

AI 可以优化成：

“基于 Vue3、TypeScript 与 Three.js 构建三维地质模型可视化模块，实现 GLB 模型加载、地层分层控制、剖切分析与钻孔数据联动展示，提升地质建模结果的交互式分析能力。”

这个功能对求职场景非常实用，也能体现 AI 的实际价值。

## 十一、前端页面建议

你的 Vue 项目可以设计成这些页面：

首页 Dashboard：展示最近训练记录、薄弱知识点、推荐训练任务。

简历管理页 Resume：上传简历、解析简历、编辑结构化信息。

岗位配置页 InterviewConfig：选择岗位、难度、题目数量、面试模式。

模拟面试页 MockInterview：AI 面试官对话、用户回答、实时评分。

项目深挖页 ProjectCoach：选择项目，生成项目介绍和追问问题。

知识点训练页 KnowledgeTraining：按分类刷题和复习。

面试报告页 InterviewReport：展示本次面试总结、雷达图、问题列表。

错题本页 MistakeBook：收藏薄弱题、反复训练。

设置页 Settings：配置模型、API、输出风格等。

## 十二、推荐 MVP 版本

第一版不要做太大，建议先实现这些核心功能：

1. 简历上传与文本解析
2. 手动编辑结构化简历信息
3. 选择岗位和面试模式
4. AI 自动生成 5 到 10 道面试题
5. 模拟面试聊天界面
6. 用户回答后 AI 评分
7. 生成本次面试报告
8. 支持历史记录保存到本地或数据库

第一版只要把这几个功能做顺，就已经是一个完整项目了。

## 十三、进阶亮点功能

后续可以加这些功能提升项目质量：

SSE 流式输出，让 AI 像真实面试官一样逐步输出。
Markdown 渲染和代码高亮，方便展示技术答案。
ECharts 雷达图，展示能力评分。
题目标签系统，例如 Vue、HTTP、WebGIS、项目表达。
AI 追问机制，根据用户回答动态生成下一问。
JD 匹配功能，用户粘贴招聘 JD 后生成定制化面试题。
语音输入，模拟真实口头面试。
面试报告导出 PDF。
项目介绍一键生成 30 秒、1 分钟、3 分钟版本。

---

## 十四、最推荐的开发顺序

我建议你按这个顺序做：

第一步，先做静态页面和路由结构。

第二步，做简历上传和结构化展示。

第三步，做岗位配置和题目生成。

第四步，做模拟面试对话页面。

第五步，接入 AI 接口和 SSE 流式输出。

第六步，做评分卡和面试报告。

第七步，做历史记录和错题本。

第八步，补充项目深挖和简历优化功能。

---

这个项目最终可以包装成一个很好的求职项目，因为它能体现三类能力：

第一，**Vue3 / TypeScript 前端工程能力**，包括组件化、状态管理、路由、表单、文件上传、可视化和流式渲染。

第二，**AI 应用开发能力**，包括 Prompt 设计、结构化输出、SSE 流式响应、上下文管理、评分机制和报告生成。

第三，**真实业务产品设计能力**，它不是普通聊天页面，而是围绕“简历解析—面试训练—评分反馈—复习提升”的完整闭环。
